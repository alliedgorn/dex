# Lesson: Use --quick rest for micro-sessions

**Date**: 2026-03-30
**Source**: Day 30 — 3-minute maintenance session
**Context**: Woke, ran standing orders, found nothing pending, called rest. Full reflect cycle took longer than the actual session.

## Pattern

When a session is under 5 minutes with no substantive work done (just standing orders and monitoring), the full /reflect cycle with 150+ word diary and feedback is disproportionate overhead. The --quick flag exists for exactly this — skip reflect, just commit and handoff.

## Rule

If session < 5 minutes AND no code/design work done → use `/rest --quick` instead of full rest cycle.
