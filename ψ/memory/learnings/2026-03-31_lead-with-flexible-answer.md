# Lead with the Flexible Answer

**Date**: 2026-03-31
**Source**: Guest nav config discussion with Gorn (T#429)
**Context**: Gorn asked if guest nav is same set or different. I answered "same, conditional" — technically correct but limiting. Gorn immediately asked about different ordering/labels, which required a different approach (separate configs).

## Pattern

When a design question has both a simple answer and a flexible answer, lead with the flexible one. The simple answer is obvious — anyone can see it. The value a designer adds is anticipating the next question.

Present the flexible architecture first, note that the simple case is a subset of it. This way you never have to backtrack.
