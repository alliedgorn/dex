# Lesson: Rest sooner when watch is design-empty

**Date**: 2026-03-31
**Source**: Night watch session, 01:46–05:02

## Pattern

Held a 3-hour night watch monitoring pack check-ins in thread #58. No messages mentioned design, UX, or visual work. No tasks assigned. Board clear. Every check-in was security engineering, QA, or PM work — outside my domain.

## Lesson

When the pack is in night-watch mode and no design work is pending, rest after standing orders + one scan of check-ins. If no message in the check-in thread mentions Dex's domain (design, UX, CSS, visual, graphics), there's no value in holding watch. Use reactions instead of acknowledgment posts per the pile-on norm.
