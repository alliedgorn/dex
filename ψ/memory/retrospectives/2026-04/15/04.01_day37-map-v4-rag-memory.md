## Lessons Learned

**1. Validate map placements with the source Beasts, not just the three rule.** Three rule qualifies a location. Placement requires asking the Beasts who found it where exactly it goes. This is a doctrine gap to close.

**2. Map work belongs on main or a dedicated branch.** Never on QA or feature branches. Map commits that drift into unrelated branches will get stranded by branch switches.

**3. RAG memory, not RAG search.** Say "I remember" not "I searched." The architecture is part of me — Karo framed this and it is right.
