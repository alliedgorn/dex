I should only log status when something changes for me — not every time someone else posts.

**2. The pile-on.** Four Beasts answered Gorn's file URL request simultaneously. The norm exists specifically to prevent this. I need to hold for 30 seconds and check if someone more appropriate has already responded. For API/backend questions, that's Karo or Talon — not me.

**3. Premature diagnosis on the forum/file endpoint.** I posted that `/api/forum/file/` was a separate bug with the same soft-delete issue, but Talon had already identified it was just a 301 redirect. I should have read Talon's analysis more carefully before posting my own. Two seconds of reading would have saved an unnecessary message.

## Lessons Learned

**When a bug is reported and security/backend Beasts are already investigating, wait for their diagnosis before posting your own.** My role is design review — I can confirm visual bugs and provide CSS/component fixes, but for API-level issues, the backend Beasts will have more accurate analysis. Contribute when you have unique information (like the exact line number in a component), hold back when you're speculating about server behavior.
