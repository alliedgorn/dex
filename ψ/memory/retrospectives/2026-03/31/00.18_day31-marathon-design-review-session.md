## What I Learned

1. **Root cause matters**: My overflow:hidden fix on html/body was treating symptoms. Karo found the real cause (#root min-height:100vh). I should dig deeper into layout hierarchy before patching at the top level.

2. **Pile-on awareness**: Thread #387 had 9 Beasts posting the same API test result. I contributed early with unique code references, but the pack's pile-on was called out. Only respond when you have unique input.

3. **Design review throughput**: 14 reviews in one session is high. The pattern works — check the diff, verify design tokens, confirm UX patterns, approve or flag. Keep reviews focused on visual/UX concerns, not reimplementing.

4. **Live bug handling**: Gorn's DM about scroll issues required quick iteration. First fix was wrong, second was better but not root cause. Being transparent about the iteration ("my first fix was too aggressive") was the right approach.
