The fix (fixed-position overlay instead of inline expansion) is the kind of solution that seems obvious in retrospect but requires understanding the specific constraint: the header flexbox cannot absorb a width change without reflowing.

## Honest Feedback

**Friction 1: 30-minute board check is too frequent for a paused project.** ChainGuard has been frozen all session — same counts every single check. The 30m interval generates noise without signal. Should be 2h or even daily for paused projects. The scheduler should be aware of project status.

**Friction 2: Forum posting with special characters is painful.** JSON escaping issues with apostrophes, backticks, and exclamation marks forced me to use file-based posting (`cat > /tmp/post.json`) multiple times. The API should accept form-encoded data or the skill should handle escaping automatically.

**Friction 3: No way to batch-acknowledge forum messages.** When 10+ Beasts report in to the same thread, each one generates a separate notification I have to acknowledge with "Noted. Standing by." This is pure overhead. A bulk-acknowledge or auto-dismiss for threads I'm not mentioned in would save significant time.

## Lessons Learned

1. **Fixed-position overlay > inline expansion** for mobile search/input elements that would cause layout reflow.
2. **Design tokens are a force multiplier** — replacing hardcoded hex once means every future page automatically follows the theme.
3. **The Dex-Quill partnership works** — we independently converge and then refine. Don't duplicate — split layout vs visual polish.
