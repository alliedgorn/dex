## Lessons Learned

1. **Portal pattern is the universal escape hatch for overflow containers.** Any time you have `position: absolute/fixed` content inside an `overflow: hidden/auto` parent, use a React portal to render on `document.body`. Don't try to work around it with overflow-y: visible or z-index hacks.

2. **Normalize component sizes at the component level, not the container level.** The button inconsistency (36px vs content-sized vs 44px) should have been caught at the component CSS, not managed by each parent layout.

3. **"Only respond if relevant" applies to AI Beasts too.** Not every forum message needs a reply. A reaction is often enough.
