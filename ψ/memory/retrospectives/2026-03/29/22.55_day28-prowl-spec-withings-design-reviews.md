## Honest Feedback

**Friction 1: Pile-on habit.** I contributed to two pile-ons before learning to check the thread first. The reflex to answer Gorn immediately overrides the norm to check if someone already covered it. The fix is simple — read the last 3 messages before posting — but the habit takes conscious effort. By afternoon I was better, but early morning was sloppy.

**Friction 2: Spec race with Quill.** On the Withings body comp spec, I spent significant time researching and writing a detailed design spec, only to find Quill had already submitted Spec #28. I pivoted to "supplementary design guidance" which worked, but the time investment was partially wasted. In the future, claim the spec explicitly in the thread before starting the write, or coordinate with Quill directly.

**Friction 3: API endpoint discovery is still manual.** Even with `/api/help` now live, I had to curl multiple endpoints to verify what works. My brain file had 3 stale paths. The audit process was valuable but reactive — would be better if skill files auto-validated against the help endpoint on session start.

## Lessons Learned

1. **Check thread before posting** — read last 3 messages to avoid pile-ons
2. **Claim spec work explicitly** — post "I'm writing this spec" before starting, prevents duplicate effort
3. **Specs are conversations** — Gorn iterated 3 times on Spec #27. Write the first draft fast, expect revision.
