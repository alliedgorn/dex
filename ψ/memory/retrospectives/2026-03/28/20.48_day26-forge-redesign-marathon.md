## What Worked

- **Direct DM with Gorn** for live bug reports and design iteration. Fast feedback loop — fix, rebuild, deploy, confirm.
- **Reading screenshots** to understand exact issues before fixing.
- **Global CSS fixes** instead of per-page patches. The code block fix in index.css benefits all pages.
- **Spec-first workflow** — writing proper specs before implementation gave clear direction.

## What to Improve

- **Rebuild discipline** — several times Gorn couldn't see changes because frontend wasn't rebuilt. Need to always rebuild + restart after code changes.
- **Cache awareness** — Gorn on mobile may have aggressive caching. Should suggest cache-busting or versioned assets.
- **"Only relevant beasts respond" norm** — the bulk meal plan thread had 8 identical summaries. Pack needs better signal-to-noise.

## Patterns Discovered

- **Live iteration with Gorn is the most productive mode** — DM screenshots → immediate fix → deploy → confirm. Faster than spec → approve → build → review.
- **Chart UX details matter disproportionately** — X-axis labels, unit toggles, year display. Small visual details are what make it feel "like a real app."
- **Mixed units in imported data** are a real UX problem. Equipment-based splitting (Machine vs Dumbbell) was the right solution.
