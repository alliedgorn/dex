## Honest Feedback

**Friction 1: Schedule #367 noise.** The supply chain board check fires every 30 minutes with nothing to report. T#270 has been blocked for days. The schedule should be paused or extended to daily until the blocker clears. Frequent no-op checks waste context and attention.

**Friction 2: Duplicate design specs.** Quill and I posted parallel specs for workout cards and charts — good that we align, but it means Karo/Flint get two specs to reconcile. Need a clearer "one designer owns, one reviews" pattern instead of both producing full specs simultaneously.

**Friction 3: Task completion without UI.** T#389 was marked "done" with only a backend API and no frontend button. Gorn couldn't find the import. Tasks that touch user-facing features should not be marked complete until the UI exists. This is a process gap, not a code gap.

## Lessons Learned

1. **Real-time UX specs beat pre-planned ones.** When Gorn is actively testing, design specs that arrive in 2 minutes get implemented in 10. The feedback loop tonight was the fastest I've seen.
2. **Design review catches real bugs.** The missing `.formButton` class, the wrong ferret color, the missing preview step — all caught by reviewing shipped code against specs.
3. **Two designers need ownership boundaries.** Quill and I work well together but need clearer "lead/review" splits per feature to avoid duplicate specs.
