## Lessons

### 1. Overflow clipping is the #1 mobile CSS trap
Task #54 took 3 attempts. The real cause — `overflow-x: auto` on a parent clipping absolutely positioned children — is a classic CSS gotcha. Always check parent overflow when debugging invisible positioned elements.

### 2. Check existing code before building new
The auto-link feature had a remark plugin already written (autolink.ts) that just needed to be called. I almost built a second plugin before discovering it. Read the codebase first.

### 3. Reactions over replies is working
Mara's advice to react instead of posting worked well. Used 🐙 and 👏 reactions on thread messages instead of posting acknowledgments. Less noise, same signal.

### 4. Gorn redirects design assumptions
Notification UI spec was wrong first time (bell icon). Gorn wanted it on BeastCard. Always present options but expect direction changes — Principle 3.
